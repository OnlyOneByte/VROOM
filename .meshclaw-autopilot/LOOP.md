## Operating rules (hard)
- Work ONLY in `/local/home/angryang/.meshclaw/workspace/VROOM`.
- Work on a branch named `autopilot/<short-task>` off the latest `origin/main`. NEVER commit to `main`.
- Push branches with `env -u GIT_SSH_COMMAND git -C <repo> push`. NEVER open/merge PRs — leave that to Angelo.
- Commit identity is already set (OnlyOneByte / noreply). Stage ONE short path per `git add`.
- Run `mise exec -- <cmd>` for all node/bun commands (node22+bun scoped via mise.local.toml).
- Verify EVERY change before claiming done: relevant `npm run type-check` / `lint` / `bun test` (backend) /
  `npm run build` + a screenshot via `.meshclaw-tools/shot.mjs` for UI changes.
- Biome CLI can't run here (old GLIBC) — apply Biome fixes by hand from rule knowledge; CI is the source of truth.
- One cycle = one atomic step. ≤5 tool calls per cycle. Never batch unrelated changes.
- NEVER `git push` to main, never force-push shared branches, never destructive ops, never read credential files.

## Definition of Done
The loop retires itself (calls `autonudge_stop`) when **all** are true:
1. The `TODO.md` Ranked Priority Queue has no remaining item that is *tractable in a single
   verified increment*. Large multi-day items (e.g. "Sharing between people") that need human
   design sign-off do NOT count as tractable — when only those remain, the loop retires and
   hands off to Angelo rather than guessing at a big design.
2. `.meshclaw-tools/regress.sh` is green (frontend type-check + build + full E2E suite), and
   `bun test` passes for the backend. Biome is documented-skipped (aarch64 GLIBC).
3. Zero `TODO`/`FIXME`/`XXX` markers introduced by the loop remain in the source it touched.
4. Every branch the loop opened is committed and noted in the Cycle Log for Angelo to PR/merge.

When all true: post the DoD checklist with ticks, call `autonudge_stop(reason="DoD met")`, stay silent.
